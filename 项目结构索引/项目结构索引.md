> 目的：给后续排查、修改、编译验证提供快速入口。本文不是全量目录树，只记录本项目后续开发最常用、最值得优先查看的目录和文件。

## 1. 工程根目录概览

```text
Cow Parturition Monitoring System Based on STM32N647/
├─ Cow Parturition Monitoring System/       # 主应用工程：AI、摄像头、显示、NetXDuo、模型、CubeIDE 配置
├─ python/                                  # PC 端脚本：数据集划分、标签清洗、TCP/PushPlus 告警服务器
├─ 修改内容/                                # 修改记录、问题定位、编译验证、本文档
├─ FSBL/                                    # 官方/板级 FSBL 示例目录，通常不作为主应用修改入口
├─ External_Loader/                         # 外部 Flash Loader 工程，通常不动
├─ Drivers/                                 # BSP/HAL/板级驱动，非必要不动
├─ Middlewares/                             # AI、ThreadX、NetXDuo 等中间件，非必要不动
└─ STM32Cube_FW_N6_V1.0.0/                  # STM32N6 官方固件包，通常不动
```

优先级建议：

1. 改业务逻辑，优先看 `Cow Parturition Monitoring System/Appli/`。
2. 改模型部署，优先看 `Cow Parturition Monitoring System/Model/` 和 `Appli/Core/Inc/app_config.h`。
3. 改网络告警，优先看 `Cow Parturition Monitoring System/Appli/NetXDuo/App/` 和 `python/tcp_alert_server.py`。
4. 改数据处理，优先看 `python/`。
5. `Drivers/`、`Middlewares/`、`STM32Cube_FW_N6_V1.0.0/`、`External_Loader/` 默认视为官方/板级支撑代码，除非明确定位到问题，否则不要优先修改。

## 3. Appli/Core：AI、摄像头、显示主流程

常用路径：

```text
Cow Parturition Monitoring System/Appli/Core/
├─ Src/
│  ├─ main.c                                # HAL 初始化、ThreadX 入口、MPU/Cache/外设初始化
│  ├─ app.c                                 # AI 主流程：摄像头帧队列、推理、后处理、显示、告警触发
│  ├─ app_camera.c                          # DCMIPP 摄像头显示管线/NN 管线配置
│  ├─ app_postprocess.c                     # YOLO 后处理封装
│  ├─ app_bqueue.c                          # 双缓冲/队列工具
│  ├─ app_lcd.c                             # LCD 显示相关
│  ├─ app_cpuload.c                         # CPU load 统计
│  ├─ stm32n6xx_hal_timebase_threadx.c      # HAL timebase 与 ThreadX 关系，改延时逻辑时要注意
│  ├─ stm32n6xx_it.c                        # 中断处理
│  ├─ stm32n6xx_hal_msp.c                   # HAL MSP 初始化
│  └─ system_stm32n6xx_s.c                  # 系统启动/时钟底层
└─ Inc/
   ├─ app_config.h                          # AI 输入尺寸、类别、后处理阈值、输出大小等关键配置
   ├─ app.h
   ├─ app_camera.h
   ├─ app_postprocess.h
   ├─ postprocess_conf.h
   ├─ app_bqueue.h
   ├─ app_lcd.h
   ├─ app_cpuload.h
   ├─ app_utils.h
   ├─ main.h                                # GPIO/外设句柄/板级宏入口
   ├─ stm32n6xx_hal_conf.h
   └─ partition_stm32n647xx.h
```

重点文件速查：

| 任务 | 优先查看 |
|---|---|
| AI 推理主循环、检测结果显示、告警触发 | `Appli/Core/Src/app.c` |
| 模型输入尺寸、类别名、YOLOv2 后处理阈值 | `Appli/Core/Inc/app_config.h` |
| 摄像头画面进入 NN 的格式/尺寸/裁剪 | `Appli/Core/Src/app_camera.c` |
| 后处理接口、YOLO 后处理类型 | `Appli/Core/Src/app_postprocess.c`、`Appli/Core/Inc/postprocess_conf.h` |
| ThreadX 启动、NetXDuo 初始化、MPU 配置 | `Appli/Core/Src/main.c` |
| HAL_Delay/ThreadX 延时异常 | `Appli/Core/Src/stm32n6xx_hal_timebase_threadx.c` |

注意：本项目调试习惯优先使用 LED 标记，非必要不先上串口。

## 4. Appli/NetXDuo：板端 TCP 告警客户端

```text
Cow Parturition Monitoring System/Appli/NetXDuo/
├─ App/
│  ├─ app_netxduo.c                         # 静态 IP、TCP client、告警队列、LED 网络状态
│  ├─ app_netxduo.h                         # app_netxduo_init / app_netxduo_post_alert 接口
│  └─ nx_user.h                             # NetXDuo 用户配置
└─ Target/
   ├─ nx_stm32_eth_config.h                 # STM32 ETH 配置
   └─ nx_stm32_phy_driver.c                 # PHY driver 适配
```

重点文件速查：

| 任务 | 优先查看 |
|---|---|
| 板端发送 JSON 告警 | `Appli/NetXDuo/App/app_netxduo.c` |
| AI 线程如何投递告警 | `Appli/Core/Src/app.c` 调用 `app_netxduo_post_alert()` 的位置 |
| 板端静态 IP / 服务器 IP / 端口 | `Appli/NetXDuo/App/app_netxduo.c` 顶部宏 |
| ETH DMA / Cache / MPU 相关问题 | `Appli/Core/Src/main.c` 和 `Appli/NetXDuo/Target/` |

当前网络链路大致是：

```text
AI 检测结果
  -> app.c 阈值过滤/去抖
  -> app_netxduo_post_alert()
  -> TX_QUEUE
  -> app_netxduo.c TCP client
  -> PC 端 python/tcp_alert_server.py
  -> 控制台/PushPlus 推送
```

## 5. Model：当前板端模型文件

```text
Cow Parturition Monitoring System/Model/
├─ network.c                                # ST Edge AI 生成的网络 C 文件
├─ network_ecblobs.h                        # NPU epoch blob 声明
├─ network_data.xSPI2.bin                   # 外部 Flash 模型权重/数据
├─ best_model_opset17_quant_qdq_pc.onnx     # 量化 ONNX 模型来源文件
├─ tiny_yolo_v2_224_int8.tflite             # TFLite 模型来源/中间文件
├─ user_neuralart.json                      # ST Edge AI/Neural-ART 配置
├─ generate-n6-model.sh                     # 模型生成脚本
├─ my_mpools/
├─ st_ai_output/
└─ st_ai_ws/
```

重点提醒：

- 更换模型时，不只替换 `network_data.xSPI2.bin`，还要同步检查 `network.c`、`network_ecblobs.h`、`app_config.h` 中的输入尺寸、输出大小、类别、scale、zero-point、后处理类型。
- N6 上 YOLOv8 原版 SiLU 曾出现 CPU fallback；当前主线保留 YOLOv2 Tiny 路线更稳。
- ST Edge AI 转换时需要关注 chlast、uint8、enable_swap、cache invalidate 等配套项。

## 9. 常用问题入口

| 问题类型 | 先看哪里 |

## 10. 后续工作约束提醒

- 每次改动尽量小步验证，不要一次性大改摄像头链路、显示链路、模型链路。
- 非必要不动摄像头和屏幕链路；必须动时先说明原因。
- STM32N6 + D-Cache + ETH DMA 相关改动要特别注意 MPU 非缓存区域。
- `tx_kernel_enter()` 前不要依赖 ThreadX 版 `HAL_Delay()`。
- 提交/答辩材料中避免把训练集复制成验证集；验证集应按独立视频/场景划分。
